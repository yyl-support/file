MoRe: Motion-aware Feed-forward 4D Reconstruction Transformer

Juntong Fang, Zequn Chen, Weiqi Zhang, Donglin Di, Xuancheng Zhang, Chengmin Yang, Yu-Shen Liu 清华大学软件学院 · 理想汽车 Li Auto Submitted 5 Mar 2026 · v2 6 Mar 2026

TL;DR

  1. 问题：单目视频的 4D 重建中，运动物体会"污染"相机位姿估计；现有方案要么训练数据以静态为主，要么依赖多阶段优化 + 辅助光流/分割，难以实时与流式部署。
  2. 方法：MoRe 是 feed-forward + 流式 的 4D 重建 Transformer。三招：Attention-Forcing（训练时把 camera token 的注意力对齐到 GT 运动掩码，推理时零开销）、Grouped Causal Attention（同帧双向 + 跨帧因果，配合 KV cache 支持流式）、BA-like 精修（序列结束再做一次全局 token 聚合，相当于一个轻量 BA）。
  3. 结果：在 Sintel / TUM-dynamics / Bonn / KITTI 上 Pose & Depth 全面 SOTA（特别在 streaming 类方法中），KITTI 512×144 上 30.09 FPS（A800），论文称速度/质量 Pareto 优于同期 Stream3R / CUT3R / WinT3R / StreamVGGT / Spann3R。

1. 背景与动机

4D 重建的目标是同时恢复每帧相机位姿 {gₜ∈ℝ⁹} 与稠密几何 {Dₜ, Pₜ}，是 AR/VR、机器人、数字孪生、沉浸式内容创作的基础模块（论文 §1）。

两条主流路线，各有痛点

经典几何法 (SfM/MVS/SLAM)

在受控静态环境下精度高，但一旦物体形变、运动或相机大范围运动会显著退化；动态 SLAM 需额外分割/光流管线。

Feed-forward 深度法 (DUSt3R/MASt3R/MapAnything/VGGT/Fast3R)

单次前向得到 pose + depth，泛化强；但训练数据以静态为主，遇到运动物体会把"运动"误当"相机"建模，且注意力随序列长度二次增长，难做流式。

混合优化管线 (MonST3R / Align3R / MegaSaM / Uni4D)

集成深度 + 光流 + 运动分割 + 迭代 BA，能处理动态但多阶段、计算贵、难实时。

流式方法 (CUT3R / Stream3R / WinT3R / StreamVGGT)

借用 LLM causal attention + KV cache 支持长序列；缺点是标准 causal mask 把同一帧内 token 也压成 flat 序列，破坏帧内空间对应，且误差会累积漂移。

MoRe 的核心问题陈述：能否设计一个 fast + generalizable 的框架，同时 处理相机和物体运动，支持流式/长序列，并给出准确的 pose + depth —— 且在推理时不依赖任何显式运动先验。

2. 核心贡献

  1. 统一框架 MoRe，联合预测相机位姿、深度、点图和运动掩码，支持动态 4D 场景。
  2. Attention-Forcing 策略：训练期显式监督 + 隐式几何一致性，把"动 vs 静"从特征空间中分离，推理期完全 test-time-free。
  3. 时序感知流式推理：Grouped Causal Attention + BA-like token 聚合，兼顾长时依赖与实时性。
  4. 在多个 benchmark 上取得 SOTA，验证零样本泛化能力。

3. 方法详解

3.1 问题形式化

给定单目视频 {Iₜ∈ℝ^(3×H×W)}, t=1..T，目标输出每帧的 {Dₜ, gₜ, Pₜ, Mₜ}，即 流式地联合估计 depth / camera (9D, 含内外参+尺度) / 动态点图 / 运动掩码 (论文 Eq.2)：

{Dₜ, gₜ, Pₜ, Mₜ}ₜ₌₁ᵀ = f_θ( {Cₜ}ₜ₌₁ᵀ⁻¹, I_{T-1} )

其中 Cₜ 是第 t 帧的缓存信息。运动掩码在训练时是 GT，推理时由模型自己输出，不再依赖外部 mask。

3.2 Attention-Forcing (核心创新 ①)

动机：VGGT 为什么在动态场景会失败？

论文 §3.2 给出关键观察：把 VGGT 的 camera token 注意力图可视化后，当场景中存在运动物体时，camera token 的注意力会"被动态区吸走"，均匀地散布到整个图像上（论文 Fig. 3 右），导致相机估计被严重污染。

做法

把 GT 运动掩码 Mₜ 按 patch size s×s 切成 mask token {mᵢ}，定义每个 image token 的"运动得分"（论文 Eq.3）：

aᵢ = 1 − (1/s²) · Σ_{(u,v)∈mᵢ} mᵢ(u,v)   ∈ [0,1]

aᵢ 越大表示该 token 越"静"。然后在训练时用 Eq.(8) 的 motion-gated 损失约束 camera token 的注意力分布 {αᵢ}：

L_attn = (1/M) · Σᵢ max(0, aᵢ − C) · αᵢ

关键设计：阈值 C 让"静态区"的 token 不被强制参与 loss 分配，动态区的 token 才被监督"压低注意力"，避免 KL-based 损失在静态场景下引入的归一化偏置（论文附录 §9.3 给出 KL vs motion-gated 的消融对比）。

推理时完全 test-time-free：没有引入任何额外输出头或后处理。MoRe 仍然以原始端到端形式输出 pose/depth/point-map/mask，但内部特征已"学会"忽略动态物。

3.3 Grouped Causal Attention (核心创新 ②)

为什么不能直接搬 LLM 的 causal mask？

LLM 把 token 当成平坦序列，用上三角 causal mask 强制自回归。但图像 token 是 2D 空间结构，同一帧内的 token 必须互相看见才能维护空间一致性。

MoRe 的解法：Frame-wise Causal Mask —— 跨帧因果，同帧内全连接（论文 Fig. 4）。

流式推理 + KV cache

首帧初始化 KV cache，之后每帧 Iₜ 只需与历史 KV 拼接后做 attention（论文 Eq.4）。配合 sliding window（论文 §8.1 提及，参考 StreamingLLM / Attention Sink）避免 KV 无限增长。

3.4 BA-like 精修 (核心创新 ③)

严格因果下 camera token 只能看"自己 + 过去"，长序列会丢失全局一致性。MoRe 的解法很朴素：

  1. 流式阶段先快速得到一份 camera token Qₜᶜᵃᵐ 与每帧 KV；
  2. 序列结束后，对每个 camera token 再跑一次 跨全部 [K₁:ₜ, V₁:ₜ] 的 attention（论文 Eq.5），相当于一个轻量 BA，不增加额外监督，仅靠特征对齐修正累积漂移。

训练时通过 复制 camera token 显式监督两路（streaming 路径 + 聚合路径）的相对位姿（论文 §3.4 末尾），其中复制 token 全时序保留梯度，原始 token 早期帧 detach，避免 BPTT 过深。

6. 个人点评

6.1 优势

  • test-time-free 解决动态污染：Attention-Forcing 是个非常优雅的 trick —— 没引入新的 head、没引入光流/分割的额外监督，推理时单次前向，且在 Sintel/Bonn/TUM-dynamics 上 streaming SOTA。
  • 流式 + 长序列友好：Frame-wise causal + KV cache + sliding window，KITTI 30 FPS 实测，论文给出多种长序列定性结果。
  • 端到端多任务：depth / pose / point-map / motion-mask 统一一个 Transformer，无需分阶段管线。
  • 训练成本相对友好：64×A800 × 2 天，对 12 个数据集的覆盖范围而言，门槛并不夸张。

6.2 局限 (论文 §10 直接总结)

  • 依赖运动掩码标注：Attention-Forcing 训练时强依赖 GT mask 质量，论文自建了一个 SAM2 + SEA-RAFT 的伪标签管线，但本质上仍是"伪监督"。
  • 长时依赖仍有限：流式 + sliding window 对极长序列的全局一致性仍是近似方案；BA-like 精修是事后的、不能用于在线场景。
  • 极快/非刚体运动：rapid motion + motion blur 下，attention alignment 不可靠（论文原文已点出）。
  • 遮挡与外观剧变：当前未显式处理；4D 一致性在严重遮挡下仍会出 artifact。

6.3 一些延伸思考

  • 与 π³ / VGGT 的关系：MoRe 选择把"运动感知"做成训练时的隐式偏置，而不是网络架构上的硬分支。这条路线在工业部署里通常更受欢迎 —— 推理 pipeline 不变。是否会在更大 backbone（如 7B 级 VLM）上依然有效，值得后续工作验证。
  • 关于 attention-forcing 损失的扩展性：motion-gated 损失 (Eq.8) 实际是一个通用"先验对齐"框架 —— 未来可被光流、不确定性、深度边界等替代监督替换，把"软知识"注入 Transformer。
  • BA-like 精修的工程含义：它把 BA 思想（多次最小化重投影误差）替换为"额外一次 attention"，从数值优化变成了特征空间内的近似 BA。在 streaming 框架下这种替换尤其划算。
  • 对 AR/AR 眼镜的潜在影响：30 FPS 流式 + test-time-free 极贴近 AR 眼镜的算力/功耗预算，是工业落地最有说服力的组合。

6.4 一句话总结

MoRe 的核心是把"动态物会污染相机估计"这件事从数据分布问题，变成一个可监督的注意力先验；再用 frame-wise causal 把这套方法低成本地搬到流式 + KV cache 框架里。对想做 4D 重建落地的工程团队，这是 2026 年最值得精读的一篇。
