## 三、Issue #921 测试（RAG API）

### 3.1 认证

```bash
# 无 token → 401
curl -s -o /dev/null -w "%{http_code}" -X POST $TEST_DOMAIN/api/v1/rag/retrieve \
  -H "Content-Type: application/json" -d '{"query":"test"}'
# 预期: 401

curl -s -X POST $TEST_DOMAIN/api/v1/rag/retrieve \
```

### 3.4 限流

```bash
for i in $(seq 1 105); do
  code=$(curl -s -o /dev/null -w "%{http_code}" $TEST_DOMAIN/api/v1/rag/documents/status_counts \
    -H "Authorization: Bearer $TOKEN")
  echo "$i: $code"
done
# 第 101 次起预期: 429
```

### 3.5 错误码

| 场景 | 预期 |
|------|------|
| 无 token | 401 `TOKEN_MISSING` |
| 假/过期 token | 401 `TOKEN_INVALID` |
| 空请求体 | 400 `INVALID_REQUEST` |
| 单用户超过 100次/小时 | 429 `RATE_LIMITED` |

### 3.6 OIDC 回调安全

| 场景 | 预期 |
|------|------|
| 无 code 参数 | 400 |
| 无 state 参数 | 400 |
| 伪造 state | 400 `INVALID_STATE` |

### 6.2 token 刷新流程

```bash
# 用 refresh_token 换新 token
REFRESH_TOKEN="<auth/callback 返回的 refresh_token>"
curl -s -X POST $TEST_DOMAIN/api/v1/rag/auth/refresh \
  -H "Content-Type: application/json" \
  -d "{\"refresh_token\":\"$REFRESH_TOKEN\"}"
# 预期: 200，返回新 access_token

# 用旧 refresh_token 再次刷新 → 应失败
curl -s -o /dev/null -w "%{http_code}" -X POST $TEST_DOMAIN/api/v1/rag/auth/refresh \
  -H "Content-Type: application/json" \
  -d "{\"refresh_token\":\"$REFRESH_TOKEN\"}"
# 预期: 400（refresh_token 仅一次有效）
```

### 6.3 OIDC 回调端点安全

```bash
# POST 方法访问 GET 端点
curl -s -o /dev/null -w "%{http_code}" -X POST $TEST_DOMAIN/api/v1/rag/auth/callback \
  -H "Content-Type: application/json" -d '{"code":"fake"}'

# 无授权码直接访问 /authorize 是否正常跳转
curl -s -o /dev/null -w "%{http_code} %{redirect_url}" $TEST_DOMAIN/api/v1/rag/auth/authorize
```

## 八、一键验收脚本

```bash
#!/bin/bash
set -e
DOMAIN="${1:?Usage: $0 <test_domain>}"
TOKEN="${2:?Usage: $0 <test_domain> <access_token>}"
PASS=0; FAIL=0

check() { local code=$1 msg=$2 item=$3
  if [ "$code" = "200" ] || [ "$code" = "401" ] && [[ "$item" =~ 401 ]]; then
    echo "✅ $msg ($item): $code"; PASS=$((PASS+1))
  else
    echo "❌ $msg ($item): $code"; FAIL=$((FAIL+1))
  fi
}

echo "=== 基础检查 ==="
check "$(curl -s -o /dev/null -w '%{http_code}' $DOMAIN/health)" "健康检查" "health"
check "$(curl -s -o /dev/null -w '%{http_code}' $DOMAIN/metrics)" "Metrics" "metrics"
check "$(curl -s -o /dev/null -w '%{http_code}' $DOMAIN/api/v1/rag/documents/status_counts -H \"Authorization: Bearer $TOKEN\")" "status_counts" "doc-status"

echo "=== 认证检查 ==="
check "$(curl -s -o /dev/null -w '%{http_code}' -X POST $DOMAIN/api/v1/rag/retrieve -H 'Content-Type: application/json' -d '{\"query\":\"test\"}')" "无 token → 401" "no-token"
check "$(curl -s -o /dev/null -w '%{http_code}' $DOMAIN/api/v1/rag/documents/status_counts -H 'Authorization: Bearer fake123')" "假 token → 401" "fake-token"
check "$(curl -s -o /dev/null -w '%{http_code}' -X POST $DOMAIN/api/v1/rag/retrieve -H \"Authorization: Bearer $TOKEN\" -H 'Content-Type: application/json' -d '{\"query\":\"test\"}')" "真 token → 200" "real-token"

echo "=== 并发检查 ==="
RESULT=$(seq 30 | xargs -P30 -I{} curl -s -o /dev/null -w '%{http_code}' -X POST $DOMAIN/api/v1/rag/retrieve -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" -d '{"query":"test"}')
FAIL_COUNT=$(echo "$RESULT" | grep -c '500' || echo 0)
if [ "$FAIL_COUNT" = "0" ]; then
  echo "✅ 并发无 500"
else
  echo "❌ 并发有 $FAIL_COUNT 个 500"; FAIL=$((FAIL+1))
fi

echo "=== 结果: 通过 $PASS / 失败 $FAIL ==="
```
