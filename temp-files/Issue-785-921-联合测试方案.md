## 三、Issue #921 测试（RAG API）

### 3.1 认证

```bash
# 无 token → 401
curl -s -o /dev/null -w "%{http_code}" -X POST $TEST_DOMAIN/api/v1/rag/retrieve \
  -H "Content-Type: application/json" -d '{"query":"test"}'
# 预期: 401

# 假 token → 401
curl -s -o /dev/null -w "%{http_code}" $TEST_DOMAIN/api/v1/rag/documents/status_counts \
  -H "Authorization: Bearer fake123"
# 预期: 401

# 正常 token → 200
curl -s -X POST $TEST_DOMAIN/api/v1/rag/retrieve \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" -d '{"query":"test"}'
# 预期: 200
```

### 3.2 检索接口

```bash
curl -s -X POST $TEST_DOMAIN/api/v1/rag/retrieve \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"query":"BMC 固件升级流程","only_need_context":true}'
# 预期: 200，返回 LightRAG 原始响应
```

### 3.3 文档管理接口

```bash
# 状态统计
curl -s $TEST_DOMAIN/api/v1/rag/documents/status_counts \
  -H "Authorization: Bearer $TOKEN"

# 管道状态
curl -s $TEST_DOMAIN/api/v1/rag/documents/pipeline_status \
  -H "Authorization: Bearer $TOKEN"

# 分页查询
curl -s -X POST $TEST_DOMAIN/api/v1/rag/documents/paginated \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"page":1,"page_size":10}'
# 预期: 200，返回分页 JSON
```

### 3.4 限流

```bash
for i in $(seq 1 105); do
  code=$(curl -s -o /dev/null -w "%{http_code}" $TEST_DOMAIN/api/v1/rag/documents/status_counts \
    -H "Authorization: Bearer $TOKEN")
  echo "$i: $code"
done
# 第 101 次起预期: 429
```

### 3.5 错误码

| 场景 | 预期 |
|------|------|
| 无 token | 401 `TOKEN_MISSING` |
| 假/过期 token | 401 `TOKEN_INVALID` |
| 空请求体 | 400 `INVALID_REQUEST` |
| 单用户超过 100次/小时 | 429 `RATE_LIMITED` |

### 3.6 OIDC 回调安全

| 场景 | 预期 |
|------|------|
| 无 code 参数 | 400 |
| 无 state 参数 | 400 |
| 伪造 state | 400 `INVALID_STATE` |

## 四、交叉验证

### 4.1 两个功能互不干扰

```bash
# 同时访问 metrics 和 RAG API
curl -s $TEST_DOMAIN/metrics | grep 'forum_retrieval_latency_count'  # #785 指标正常
curl -s -X POST $TEST_DOMAIN/api/v1/rag/retrieve \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" -d '{"query":"test"}'         # #921 检索正常
```

### 4.2 健康检查覆盖两套功能

```bash
curl -s $TEST_DOMAIN/health/detail | python3 -c "
import json, sys
d = json.load(sys.stdin)
components = d.get('components', {})
required = ['monitor_instance', 'monitor_thread_alive', 'service_initialized',
            'oidc_client', 'lightrag_client']
for k in required:
    print(f'{k}: {components.get(k, \"NOT FOUND\")}')
"
# 预期: 所有组件为 true
```

### 4.3 RAG API 请求不影响 Metrics 采集

```bash
# 先查一次 metrics 基准值
val_before=$(curl -s $TEST_DOMAIN/metrics | grep 'forum_processed_topic_count' | tail -1)

# 发起一次 RAG retrieve
curl -s -X POST $TEST_DOMAIN/api/v1/rag/retrieve \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" -d '{"query":"test"}' > /dev/null

# 检查 metrics 是否仍可访问（没有因为 RAG 请求崩溃）
val_after=$(curl -s $TEST_DOMAIN/metrics | grep 'forum_processed_topic_count' | tail -1)
echo "Before: $val_before, After: $val_after"
```
